# Project Explanation: Sentiment Analysis to Detect and Prevent Twitter Spam

## 📌 Project Goals
The primary objective of this project is to build an end-to-end machine learning system capable of identifying spam messages with high precision while also analyzing the sentiment of the text. This dual-purpose system is essential for social media platforms like Twitter to filter malicious content and understand the emotional context of user communications.

## 🏗️ Architecture and Pipeline Design
The project is built using a clean, modular architecture, separating data handling, feature engineering, and modeling into distinct stages.

### 1. Data Loading & Preprocessing
- **Source**: SMS Spam Collection Dataset from Kaggle.
- **Loading**: Pandas with `latin-1` encoding for robust data ingestion.
- **Cleaning**: 
  - Lowercasing for uniformity.
  - URL removal to eliminate noise from external links.
  - Punctuation and special character removal.
  - Stopword filtering using NLTK to focus on meaningful words.
  - Porter Stemming to reduce words to their root forms (e.g., "winning" -> "win").

### 2. Exploratory Data Analysis (EDA)
- **Visuals**: Count plots to visualize class imbalance (Spam vs. Ham).
- **WordClouds**: Automatic generation of word clouds for spam messages, highlighting keywords like "FREE," "WINNER," and "URGENT."
- **Storage**: All visualizations are automatically saved to `results/plots/`.

### 3. Feature Extraction
- **TF-IDF Vectorizer**: Converts text to numerical format based on word importance.
- **N-grams**: Implemented unigrams and bigrams (`ngram_range=(1, 2)`) to capture word pairings, significantly improving context understanding.
- **Dimensionality**: Limited to the top 5,000 features for optimal performance.

### 4. Machine Learning Models
Three classical models are trained and compared:
- **Multinomial Naive Bayes (NB)**: High performance for text classification tasks.
- **Logistic Regression (LR)**: A strong baseline for binary classification.
- **Support Vector Machine (SVM)**: Often the most accurate for high-dimensional text data.
The best model (currently SVM) is automatically selected and saved to `models/`.

### 5. Advanced Deep Learning (BERT)
To leverage modern AI, the project integrates **BERT** (Bidirectional Encoder Representations from Transformers). 
- **Model**: `bert-tiny-finetuned-sms-spam-detection`.
- **Capability**: Unlike classical models, BERT understands the context of words bi-directionally, making it more robust against sophisticated spam.

### 6. Sentiment Analysis
Using the **TextBlob** library, the system classifies each message into:
- **Positive**: Optimistic or encouraging tone.
- **Neutral**: Factual or informational tone.
- **Negative**: Critical, aggressive, or pessimistic tone.
This adds a layer of intelligence, helping to distinguish between a "Free Offer" (Spam) and a "Gift from a friend" (Ham).

### 7. Evaluation & Metrics
The models are evaluated on a 20% test split using:
- **Accuracy**: Overall correctness.
- **Precision & Recall**: Critical for imbalanced datasets.
- **F1-Score**: The harmonic mean of precision and recall, our primary metric.
- **Confusion Matrix**: Visualizing true vs. false positives and negatives.

### 8. Interactive Web App (Streamlit)
A modern user interface allowing users to:
- Switch between **Classical ML** and **BERT** models.
- View real-time sentiment results.
- Visualize dataset insights and model performance in the sidebar.

---
This modular structure ensures that the project is scalable, easy to maintain, and ready for deployment in real-world scenarios.
